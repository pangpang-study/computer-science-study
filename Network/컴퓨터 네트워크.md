## 컴퓨터 네트워크와 TCP / IP

컴퓨터 네트워크는 여러대의 컴퓨터를 연결하여 서로 데이터를 주고받을 수 있도록 망이 사전에 구축된것을 의미한다.

LAN: 가정이나 사무실처럼 근거리에 위치한 컴퓨터들이 연결된 환경

WAN: 집 밖, 회사처럼 비교적 먼 거리에 위치한 컴퓨터들과 연결된 것



## 서버와 클라이언트

네트워크와 연결된 컴퓨터들 중 서비스를 제공하는 쪽을 SERVER라 부르고 서비스를 받는 쪽을 CLIENT라 부른다



## P2P

네트워크에 연결된 두 대의 컴퓨터가 클라이언트와 서버의 역할을 동시에 할 수 있어서 서로에게 서비스를 주거나 받을 수 있는 통신방식을 의미한다



## 패킷 교환방식

이메일, 파일과 같은 데이터를 패킷이라는 작은 단위로 분할 한 후 주고 받는다. 어디로 전달되어야 하는지 알 수 있도록 Address 정보를 가지고 있다.



## 회선 교환 방식 vs 패킷 교환방식

아날로그 방식의 유선전화, 3g 방식의 휴대전화는 **회선 교환 방식**을 사용한다. 기본적으로 1대1통신만이 가능하다.

여러상대와의 통신을 위해서 주고 받을 데이터를 쪼갠 형태인 패킷 교환 방식이 있다.



## 컴퓨터 네트워크를 구성하는 계층들

다양한 통신 장비와 프로그램이 어울려 통신을 가능하게 한다. 통신에 참여하는 장비나 프로그램이 맡은 역할의 기능을 분류하고 추상화 한것을 계층 모델이라고 한다.



어플리케이션 계층: 상대방에게 보낼 데이터를 담는다

트랜스포트 계층: 패킷단위로 전송할 데이터를 쪼갠 후 식별 가능한 정보를 추가한다

인터넷 계층: 수신측의 컴퓨터를 식별할 수 있는 정보를 추가한다

네트워크 인터페이스 계층: 하드웨어에 신호를 전달하는데 필요한 정보를 추가한다



**어플리케이션 계층**의 프로그램은 데이터의 내용을 보고 그에맞는 서비스를 처리하는데 집중하고, 데이터 전송에 대해서는 하위 계층에 위임한다!

그래서 사용자가 볼때에는 송신측, 수신측의 서버가 직접 통신하는것처럼 느끼게 된다

**트랜스포트 계층**은 데이터가 제대로 전달되게 port와같은 정보를 헤더에 담아 전송한다. 재전송같은 부분을 처리하기도 한다.

**인터넷 계층**은 데이터에 address의 정보를 붙여 목적지까지 무사히 전달하는 것이다. IP주소를 기반으로 주소정보를 탐색하고 router를 통해 전달된다.

**네트워크 인터페이스 계층**은 유선 Lan 어댑터나 무선 Lan 어댑터가 처리할 수 있는 형태로 데이터를 변환하고 이 데이터를 목적지까지 전달한다. 물리적으로 인접한 장비까지 데이터를 전달하기 위해 사용한다(이더넷 헤더를 포함함)



## 프로토콜이란

컴퓨터와 컴퓨터가 통신하는데 필요한 절차나 규약을 의미한다.



프로토콜 조합하기!

- 웹 페이지를 보게되는 경우
  - 이더넷 헤더 + IP 헤더 + TCP 헤더 + HTTP 데이터
- 인터넷으로 전화를 거는 경우
  - IEEE 802.11 헤더 + IP 헤더 + UDP 헤더 + VOIP 데이터



## 인터넷

인터넷은 TCP/IP라는 프로토콜을 기반으로 한  세계 최대의 네트워크이다.

인터넷은 중앙에서 관리하는 곳이 존재하지 않는다.



프로토콜은 이더넷이나 무선랜 같은 하드웨어는 IEEE가 웹과 관련된 것은 W3C가 맡아 관리한다
